PASO 3: INICIAR SERVICIOS
─────────────────────────

Opción A (Docker Compose - Recomendado):
  1. cp .env.example .env
  2. docker-compose up -d
  3. Esperar 10 segundos
  4. curl http://localhost:8000/health
  5. Ir a http://localhost:8000/docs

Opción B (Python Local):
  1. cp .env.example .env
  2. python3.11 -m venv venv
  3. source venv/bin/activate
  4. pip install -r requirements.txt
  5. python main.py
  6. Ir a http://localhost:8000/docs

PASO 4: VERIFICAR FUNCIONAMIENTO
────────────────────────────────
Abrir en navegador: http://localhost:8000/docs
(Swagger UI con documentación interactiva)

O hacer un test:
  curl -X POST http://localhost:8000/analyze \
    -H "Content-Type: application/json" \
    -d '{"text": "si no pagas habrá consecuencias"}'

PASO 5: LEER DOCUMENTACIÓN
──────────────────────────
1. README.md o README.txt para documentación completa
2. QUICK_START.txt para primeros pasos
3. Swagger UI (http://localhost:8000/docs) para API interactiva

  pip install -r requirements.txt   # Instalar
  python main.py                    # Ejecutar
